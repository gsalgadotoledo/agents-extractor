# Submission Workflow Rules

## Status Transitions

### Open → In Review
- Extraction must be completed (status: extracted)
- At least one representative assigned

### In Review → Quoting
- All required fields must be filled (no critical missing fields)
- Loss runs reviewed
- Representative has reviewed the extracted data

### Quoting → Quoted
- Quote document generated
- Premium calculated or estimated

### Quoted → Approved
- Manager or Director must approve
- No outstanding warnings flagged as critical

### Approved → Completed
- Policy document generated
- Final email sent to broker with policy documents attached
- All documents uploaded/generated

### Any → Needs Review
- If loss runs are present for 4+ years, automatically routes here
- Can also be manually flagged by a representative

### Any → Failed
- If extraction fails critically
- If submission is invalid or duplicate to be discarded

## Required Fields for Completion
- Insured name
- Broker email
- Coverage type
- Effective date
- At least one facility/address
- Loss runs status (present or not)

## Document Generation Rules
- **Quote**: Can be generated once extraction is complete and data is reviewed
- **Policy**: Can only be generated after approval
- **Binder**: Can be generated after quote is accepted
- **Certificate**: Can be generated after policy is created

## Email Rules
- Acknowledgment is sent automatically on receipt
- Follow-up emails for missing info can be sent by reps or the agent
- Final policy email requires approval before sending
